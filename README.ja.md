## 概要

Palette Pallyは、Material-UIのカラーパレットを簡単に作成・管理できる強力なツールです。直感的なUIと柔軟なカスタマイズオプションにより、デザインシステムのカラーパレットを効率的に設計できます。

### 特徴

- 🎨 **リアルタイムプレビュー**: 色の変更を即座に確認
- 🔧 **完全カスタマイズ可能**: レイアウト、テーマ、スペーシングを自由に調整
- 📱 **レスポンシブデザイン**: モバイル、タブレット、デスクトップに完全対応
- 💾 **自動保存**: ローカルストレージに設定を保存
- 📦 **エクスポート機能**: JSON、TypeScript、CSS形式で出力
- ♿ **アクセシビリティ対応**: WCAGコントラスト比を自動計算
- 🌙 **ダークモード**: ライト/ダークテーマの切り替え

## 主な機能

### 1. カラー管理

- **テーマカラー**: Primary、Secondary、Error、Warning、Info、Success
- **カスタムカラー**: 独自のカラーロールを追加可能
- **色バリエーション**: Main、Light、Lighter、Dark、ContrastTextを自動生成
- **グレーパレット**: 50～900、A100～A700の完全なグレースケール

### 2. UI柔軟性

- **レイアウトモード**:
  - グリッド: カードをグリッド状に配置
  - リスト: 縦一列に表示
  - コンパクト: 密集したグリッド表示

- **表示密度**: Comfortable / Standard / Compact

- **カスタマイズオプション**:
  - カードサイズ（Small / Medium / Large）
  - カラム数（1～6列）
  - スペーシング（Tight / Normal / Loose）
  - ボーダー半径（0～16px）
  - カードシャドウ（0～5段階）
  - アニメーション速度
  - フォントサイズ

### 3. カラーピッカー機能

- **複数のカラーモデル**:
  - HEX
  - RGB
  - HSL
  - Oklab

- **アクセシビリティチェック**:
  - WCAGコントラスト比の表示
  - AAA/AA/Aレベルの自動判定

### 4. エクスポート・インポート

- **エクスポート形式**:
  - JSON（完全な設定データ）
  - TypeScript（型定義付き）
  - CSS変数
  - MUI ThemeProviderコード

- **インポート機能**:
  - 既存の設定をJSON形式でインポート
  - バリデーション付き

## 使い方

### カラーパレットの作成

#### ステップ1: ベースカラーを選択

1. **Theme Colors**タブを開きます
2. 変更したいカラーカード（例: Primary）をクリック
3. カラーピッカーで色を選択、またはHEXコードを直接入力
4. バリエーション（Light、Lighter、Dark）が自動生成されます

#### ステップ2: カスタムカラーを追加

1. **Add Color**ボタンをクリック
2. カラー名を入力（例: "brand"）
3. カラータイプを選択:
   - **Theme**: 自動的にバリエーションを生成
   - **Simple**: 単一のカラーとコントラストテキスト
4. **Add**をクリック

#### ステップ3: 追加カラーを設定

1. **Additional Colors**タブを開きます
2. テキストカラー（Primary、Secondary、Disabled）を設定
3. 背景カラー（Default、Paper）を設定
4. Dividerカラーを設定
5. グレーパレット（50～900）をカスタマイズ

#### ステップ4: システムカラーを設定

1. **System Colors**タブを開きます
2. アクションカラー（Active、Hover、Selected等）を設定
3. 共通カラー（Black、White）を設定

#### ステップ5: 保存

1. 右上の**Save**ボタンをクリック
2. 設定がローカルストレージに保存されます
3. 次回アクセス時に自動的に復元されます

### UIカスタマイズ

#### プリセットの使用

1. 右上の**UI Settings**ボタンをクリック
2. **Quick Presets**セクションでプリセットを選択:
   - **Minimal**: シンプルで無駄のないレイアウト
   - **Spacious**: ゆったりとした広々レイアウト
   - **Compact**: 多くの情報を一度に表示
   - **Default**: 標準設定に戻す

#### カスタム設定

##### レイアウト設定

1. **Layout Mode**を選択:
   - **Grid**: カードをグリッド状に配置
   - **List**: 縦一列に表示
   - **Compact**: 密集したグリッド

2. **View Density**を選択:
   - **Comfortable**: 余白が大きい
   - **Standard**: 標準的な余白
   - **Compact**: 余白を最小限に

3. **Card Size**を選択:
   - **Small**: 小さいカード
   - **Medium**: 標準的なサイズ
   - **Large**: 大きいカード

4. **Desktop Columns**を設定（1～6列）

5. **Spacing**を選択:
   - **Tight**: 狭い間隔
   - **Normal**: 標準的な間隔
   - **Loose**: 広い間隔

##### テーマ設定

1. **Border Radius**を選択（0～16px）
   - 0px: 角ばったデザイン
   - 8px: 標準的な丸み
   - 16px: 大きく丸みをつける

2. **Card Shadow**を選択（0～5）
   - 0: 影なし（フラットデザイン）
   - 1～2: 控えめな影
   - 3～5: 立体的な影

3. **Animation Speed**を選択:
   - **Slow**: ゆっくりしたアニメーション
   - **Normal**: 標準的な速度
   - **Fast**: 速いアニメーション
   - **Off**: アニメーションなし

4. **Font Size**を選択:
   - **Small**: 小さい文字（12px）
   - **Normal**: 標準的な文字（14px）
   - **Large**: 大きい文字（16px）

### エクスポート・インポート

#### エクスポート

1. **Export**ボタンをクリック
2. エクスポート形式を選択:
   - **JSON**: 完全な設定データ
   - **TypeScript**: TypeScript型定義
   - **CSS**: CSS変数
   - **MUI Theme**: MUI ThemeProviderコード

3. **Copy**ボタンでクリップボードにコピー
4. または**Download**で JSONファイルとしてダウンロード

#### インポート

1. **Import**ボタンをクリック
2. 2つの方法でインポート:
   - **ファイルから**: JSONファイルを選択
   - **貼り付け**: JSONデータを直接貼り付け

3. **Import**ボタンをクリック
4. バリデーションに成功すると、設定が適用されます

## UI設定ガイド

### レイアウトパターン

#### 大規模プロジェクト向け（推奨設定）

```
Layout Mode: Grid
View Density: Comfortable
Card Size: Large
Desktop Columns: 2
Spacing: Loose
Border Radius: 12px
Card Shadow: 2
```

#### 高密度作業向け

```
Layout Mode: Compact
View Density: Compact
Card Size: Small
Desktop Columns: 4-6
Spacing: Tight
Border Radius: 4px
Card Shadow: 1
```

#### プレゼンテーション向け

```
Layout Mode: Grid
View Density: Comfortable
Card Size: Large
Desktop Columns: 2-3
Spacing: Loose
Border Radius: 16px
Card Shadow: 3
```

### パフォーマンス最適化

多数のカラーを扱う場合：

1. **Animation Speed**を**Fast**または**Off**に設定
2. **Card Shadow**を低く設定（0～1）
3. **View Density**を**Compact**に設定

### アクセシビリティ

視覚的なアクセシビリティを向上させるには：

1. **Font Size**を**Large**に設定
2. **Card Shadow**を高く設定（3～4）して区別しやすく
3. **Spacing**を**Loose**に設定して余白を確保

## トラブルシューティング

### 設定が保存されない

**原因**: ブラウザのローカルストレージが無効、またはプライベートブラウジングモード

**解決策**:
1. 通常のブラウジングモードを使用
2. ブラウザのローカルストレージを有効化
3. ブラウザのキャッシュとCookieを確認

### エクスポートが機能しない

**原因**: クリップボードAPIが利用できない、またはブラウザの権限が不足

**解決策**:
1. ブラウザを最新版に更新
2. HTTPSで接続（開発環境ではlocalhostでOK）
3. クリップボード権限を許可

### インポートエラー

**原因**: JSONフォーマットが不正、または必須フィールドが不足

**解決策**:
1. JSONの構文を確認（カンマ、括弧等）
2. 必須フィールド（colors、mode等）が含まれているか確認
3. カラーコードが正しい形式（#RRGGBB）か確認

### レイアウトが崩れる

**原因**: ブラウザの互換性問題、またはCSS変数のサポート不足

**解決策**:
1. モダンブラウザを使用（Chrome、Firefox、Safari、Edge最新版）
2. ブラウザキャッシュをクリア
3. ページをリロード
4. UI設定をリセット（**UI Settings** > **Default**）

## FAQ

### Q: Material-UI以外のフレームワークでも使えますか？

A: はい、エクスポートしたカラーパレットは、CSS変数やJSON形式で出力できるため、任意のフレームワークやライブラリで使用できます。

### Q: チームで設定を共有できますか？

A: はい、JSONエクスポート機能を使って設定をファイルとして保存し、チームメンバーとファイルを共有してインポートできます。

### Q: カラーパレットに何個までカラーを追加できますか？

A: 技術的な制限はありませんが、パフォーマンスとユーザビリティの観点から、20～30個程度を推奨します。

### Q: ダークモードとライトモードで異なる設定を保存できますか？

A: はい、それぞれのモードで異なるカラー設定を持つことができ、モードを切り替えても設定は保持されます。

### Q: UI設定は他のブラウザやデバイスで共有されますか？

A: いいえ、UI設定とカラーパレット設定はブラウザのローカルストレージに保存されるため、デバイス・ブラウザごとに個別に管理されます。設定を共有するには、JSONエクスポート/インポート機能を使用してください。

### Q: カスタムカラーの名前に制限はありますか？

A: 特別な制限はありませんが、JavaScriptの変数名として使用可能な名前を推奨します（英数字、アンダースコア、ハイフン）。

### Q: コントラスト比の基準は？

A: WCAGガイドラインに基づいています：
- **AAA**: 7:1以上（最高レベル）
- **AA**: 4.5:1以上（推奨レベル）
- **A**: 3:1以上（最低レベル）
- **Fail**: 3:1未満（アクセシビリティ不合格）

### Q: プロジェクトに貢献できますか？

A: はい！GitHubでプルリクエストやイシューを歓迎します。詳細は`DEVELOPMENT.md`を参照してください。
